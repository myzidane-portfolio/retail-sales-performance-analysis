# Retail Sales Performance Analysis (Google Sheets)

## Project Overview

This project demonstrates an end-to-end data analysis workflow using **Google Sheets**.
The goal is to analyze retail sales data and generate business insights through data cleaning, feature engineering, pivot analysis, KPI calculation, and dashboard visualization.

## Dataset

The dataset is a simulated retail sales dataset containing **250 transaction records** with the following fields:

* Order ID
* Order Date
* Customer ID
* City
* Product
* Category
* Quantity
* Price
* Cost
* Payment Method

## Data Analysis Workflow

The analysis follows a structured workflow commonly used by data analysts:

1. Raw Data Collection
2. Data Cleaning
3. Feature Engineering
4. KPI Calculation
5. Pivot Table Analysis
6. Dashboard Visualization
7. Insight Summary

## Key Metrics (KPI)

The dashboard highlights several important business metrics:

* Total Revenue
* Total Profit
* Total Orders
* Total Customers
* Average Order Value
* Overall Profit Margin

## Dashboard Analysis

The dashboard provides insights into:

* Revenue by City
* Revenue by Product
* Monthly Revenue Trend

## Key Insights

* Total revenue generated during the analysis period is **Rp2,810,450,000**.
* Total profit achieved is **Rp840,112,000**, with an overall profit margin of **29.89%**.
* **Jakarta** generates the highest revenue among all cities.
* **Headset** is the product contributing the most to total revenue.
* Sales performance fluctuates across different months, indicating variations in customer purchasing behavior.
